O que é um Quant?

Um Quantitative Analyst (Quant) é um profissional que aplica matemática avançada, estatística, teoria de probabilidade e ciência da computação para resolver problemas financeiros complexos. Trabalham em bancos de investimento, hedge funds, corretoras e fintechs.

Quant Researcher

Pesquisa

Desenvolve modelos matemáticos e estratégias de trading. Foco em alpha generation e análise estatística.

Quant Developer

Tecnologia

Implementa modelos em produção, sistemas de execução de ordens e infraestrutura de dados financeiros.

Quant Risk

Risco

Mede e gerencia risco de mercado, crédito e liquidez. Desenvolve sistemas de stress testing e VaR.

Retornos Financeiros: A Pedra Fundamental

Antes de qualquer modelo, precisamos entender como expressar variação de preços:

Retorno Simples
R_t = (P_t - P_{t-1}) / P_{t-1}
Variação percentual do preço entre dois períodos
Retorno Logarítmico (Log-Return)
r_t = ln(P_t / P_{t-1}) = ln(P_t) - ln(P_{t-1})
Preferido em finanças quant: aditivo no tempo, aproximadamente normal, sem problema de -100%
⚠ Por que Log-Returns?

Retornos logarítmicos são aditivos no tempo: r_{1,3} = r_{1,2} + r_{2,3}. Isso facilita enormemente modelagem estatística e garante que preços sejam sempre positivos no modelo.

PYTHON
import numpy as np
import pandas as pd

# Calculando retornos com pandas
prices = pd.Series([100, 102, 99, 104, 103])

# Retorno simples
simple_returns = prices.pct_change()

# Log-return
log_returns = np.log(prices / prices.shift(1))

# Retorno acumulado
cumulative = (1 + simple_returns).cumprod() - 1

# Estatísticas descritivas dos retornos
stats = log_returns.describe()
annualized_vol = log_returns.std() * np.sqrt(252)
sharpe = (log_returns.mean() * 252) / annualized_vol

Módulo 02 · Fundamentos

Matemática & Estatística Financeira

Ferramentas matemáticas essenciais: momentos estatísticos, correlação, covariância e processos estocásticos.

Os 4 Momentos Estatísticos

MomentoFórmulaSignificado Financeiro
1º — Média (μ)E[X] = Σ xᵢ/nRetorno esperado do ativo
2º — Variância (σ²)E[(X-μ)²]Risco total (volatilidade ao quadrado)
3º — AssimetriaE[(X-μ)³]/σ³Cauda longa para direita (positivo) ou esquerda (negativo)
4º — CurtoseE[(X-μ)⁴]/σ⁴Caudas pesadas (>3 = leptocúrtica — common em finanças)
⚠ Fat Tails — A Armadilha dos Mercados

Retornos financeiros têm curtose > 3 (distribuição leptocúrtica). Isso significa que eventos extremos ("cisnes negros") são muito mais frequentes do que uma Normal preveria. Modelos que assumem normalidade subestimam riscos de cauda.

Matriz de Covariância e Correlação

Para uma carteira com n ativos, a matriz de covariância Σ é fundamental para calcular risco de portfólio:

Variância do Portfólio
σ²_p = wᵀ Σ w
onde w = vetor de pesos, Σ = matriz de covariância n×n
Correlação de Pearson
ρ_{ij} = Cov(rᵢ, rⱼ) / (σᵢ · σⱼ) ∈ [-1, +1]
Mede a relação linear entre retornos de dois ativos. ρ=-1 é perfeita diversificação.
HEATMAP DE CORRELAÇÃO
💡 Diversificação

Ativos com correlação baixa (<0.5) reduzem o risco do portfólio sem necessariamente reduzir retorno. Este é o princípio da diversificação de Markowitz.

📌 Correlação ≠ Causalidade

Alta correlação histórica não implica relação causal. Em crises, correlações tendem a subir (efeito "flight to safety"), quebrando a diversificação exatamente quando ela é mais necessária.

Processo de Wiener e Movimento Browniano

A base matemática de praticamente todo modelo financeiro de preços é o Movimento Browniano Geométrico (GBM):

Equação Diferencial Estocástica (SDE)
dS = μS dt + σS dW_t
dS = variação do preço | μ = drift (tendência) | σ = volatilidade | dW_t = incremento Browniano ~ N(0, dt)
Solução Analítica (Lema de Itô)
S_t = S_0 · exp[(μ - σ²/2)t + σW_t]
Log-preços seguem distribuição Normal: ln(S_t/S_0) ~ N[(μ - σ²/2)t, σ²t]
PYTHON
# Simulando GBM — base para precificação de derivativos
import numpy as np

def simulate_gbm(S0, mu, sigma, T, dt, n_paths):
    n_steps = int(T / dt)
    Z = np.random.standard_normal((n_paths, n_steps))
    
    # Incrementos do log-preço
    log_returns = (mu - 0.5 * sigma**2) * dt + sigma * np.sqrt(dt) * Z
    
    # Produto cumulativo para obter preços
    paths = S0 * np.exp(np.cumsum(log_returns, axis=1))
    
    return paths

# Parâmetros
paths = simulate_gbm(
    S0=100,     # Preço inicial
    mu=0.10,    # Drift anual de 10%
    sigma=0.20, # Volatilidade anual de 20%
    T=1,        # 1 ano
    dt=1/252,   # Passos diários
    n_paths=1000
)

Teorema do Limite Central (TLC)

📐 Teorema Fundamental

A soma de n variáveis aleatórias independentes com média μ e variância σ² tende a uma distribuição Normal quando n → ∞, independente da distribuição original. Isso justifica o uso da Normal em muitos modelos financeiros com retornos agregados.

Value at Risk (VaR) Paramétrico

VaR Paramétrico (Normal)
VaR_{α} = μ - z_α · σ
z_α = quantil da Normal para nível de confiança α. Ex: z_{99%} = 2.326, z_{95%} = 1.645

Módulo 04 · Séries Temporais

Conceitos Fundamentais de Séries Temporais

Estacionariedade, autocorrelação, ACF, PACF e decomposição de séries temporais financeiras.

O que é uma Série Temporal?

Uma série temporal é uma sequência de observações ordenadas no tempo: {y_t, t = 1, 2, ..., T}. Em finanças: preços de ações, taxas de câmbio, juros, inflação, volatilidade implícita.

EXEMPLO DE SÉRIE TEMPORAL — PREÇO SINTÉTICO COM TENDÊNCIA + SAZONALIDADE + RUÍDO

Decomposição de Série Temporal

Modelo Aditivo

Y_t = T_t + S_t + R_t

Quando sazonalidade é constante ao longo do tempo. Adequado para séries com variação estável.

Modelo Multiplicativo

Y_t = T_t × S_t × R_t

Quando sazonalidade cresce com o nível da série. Mais comum em dados econômicos e financeiros.

🔑 Estacionariedade — O Conceito Central

Uma série temporal é estritamente estacionária se sua distribuição conjunta é invariante no tempo. Na prática, usamos estacionariedade fraca (covariância-estacionária):

  • E[Y_t] = μ (média constante no tempo)
  • Var[Y_t] = σ² (variância constante no tempo)
  • Cov[Y_t, Y_{t-k}] depende apenas do lag k, não de t
⚠ Preços vs Retornos

Preços de ativos são tipicamente NÃO-ESTACIONÁRIOS (têm tendência). Retornos (primeira diferença do log) geralmente são estacionários. Sempre use retornos em modelos estatísticos, não preços!

Testes de Estacionariedade

TesteH₀Decisão
ADF (Augmented Dickey-Fuller)Raiz unitária (não-estacionário)p < 0.05 → estacionário
KPSSEstacionáriop < 0.05 → não-estacionário
PP (Phillips-Perron)Raiz unitáriaAnálogo ao ADF, mais robusto

ACF e PACF — As Impressões Digitais da Série

A função de autocorrelação (ACF) e autocorrelação parcial (PACF) são ferramentas visuais essenciais para identificar o modelo correto:

ACF (AUTOCORRELAÇÃO)
PACF (AUTOCORRELAÇÃO PARCIAL)
AR(p)

ACF decai gradualmente. PACF corta após lag p.

MA(q)

ACF corta após lag q. PACF decai gradualmente.

ARMA(p,q)

ACF e PACF ambos decaem gradualmente.

Módulo 05 · Séries Temporais

Modelos ARIMA & SARIMA

Os modelos mais utilizados em previsão de séries temporais financeiras e econômicas.

Família de Modelos ARIMA

AR(p) — AutoRegressivo

Y_t = c + Σ φᵢY_{t-i} + ε_t

Valor atual é combinação linear de p valores passados. Parâmetro φᵢ: coeficientes autoregressivos.

MA(q) — Médias Móveis

Y_t = μ + ε_t + Σ θᵢε_{t-i}

Valor atual depende de choques (erros) passados. Parâmetro θᵢ: coeficientes de média móvel.

I(d) — Integração

ΔᵈY_t = Y_t - Y_{t-1}

Diferenciar d vezes para tornar a série estacionária. Para retornos, d=1 geralmente é suficiente.

ARIMA(p, d, q) — Forma Geral
φ(B)·Δᵈ·Y_t = θ(B)·ε_t
B = operador de backshift: B·Y_t = Y_{t-1} | p = ordem AR | d = grau de integração | q = ordem MA

Critérios de Seleção de Modelo

AIC (Akaike)
AIC = 2k - 2ln(L̂)
k = nº de parâmetros. Penaliza modelos complexos. Menor = melhor. Tende a selecionar mais parâmetros.
BIC (Bayesian)
BIC = k·ln(n) - 2ln(L̂)
Penalização mais forte pela complexidade. Favorece modelos mais parcimoniosos. Melhor para n grande.

Módulo 06 · Séries Temporais

Modelos GARCH & Volatilidade

Modelando a volatilidade condicional — o coração das finanças quantitativas de risco.

⚠ O Problema com Volatilidade Constante

Retornos financeiros exibem agrupamento de volatilidade (volatility clustering): períodos de alta volatilidade tendem a ser seguidos por alta volatilidade, e vice-versa. Modelos com σ constante falham em capturar esse fenômeno.

VOLATILITY CLUSTERING — RETORNOS SIMULADOS COM GARCH

Família ARCH/GARCH

ARCH(q) — Engle (1982)
σ²_t = ω + Σ αᵢ · ε²_{t-i}
Variância condicional depende de q erros passados ao quadrado. Primeiro modelo de volatilidade condicional.
GARCH(p,q) — Bollerslev (1986)
σ²_t = ω + Σ αᵢε²_{t-i} + Σ βⱼσ²_{t-j}
Generaliza ARCH: variância depende também de variâncias passadas. α+β < 1 garante estacionariedade.
VarianteInovaçãoUso Típico
GARCH(1,1)Modelo base, parcimoniosoPrevisão de vol, VaR
EGARCHCapta efeito de alavancagem (assimetria)Ações, retornos negativos pioram mais
GJR-GARCHAssimetria via indicador binárioEquities, índices
FIGARCHMemória longa na volatilidadeCâmbio, commodities
DCC-GARCHCorrelações dinâmicas condicionaisPortfólios multiativos
EGARCH — Efeito Alavancagem
ln(σ²_t) = ω + α|z_{t-1}| + γ·z_{t-1} + β·ln(σ²_{t-1})
γ < 0: retornos negativos aumentam mais a volatilidade (leverage effect). Não exige restrições de positividade.
PYTHON
from arch import arch_model

# GARCH(1,1) com distribuição t-Student
model = arch_model(
    returns * 100,      # Escalar para estabilidade numérica
    vol='Garch',
    p=1, q=1,
    dist='t'           # t-Student capta fat tails
)
result = model.fit(update_freq=5)
print(result.summary())

# Volatilidade condicional
cond_vol = result.conditional_volatility / 100
annualized_vol = cond_vol * np.sqrt(252)

# Previsão de volatilidade
forecasts = result.forecast(horizon=10)
vol_forecast = np.sqrt(forecasts.variance.values[-1,:]) / 100
Módulo 07 · Projeções

Previsão & Backtesting

Como avaliar corretamente a qualidade de previsões financeiras e evitar os erros mais comuns.

Métricas de Erro de Previsão

MAE
MAE = (1/n)Σ|yₜ - ŷₜ|
Erro absoluto médio. Interpretável na unidade original. Robusto a outliers.
RMSE
RMSE = √[(1/n)Σ(yₜ-ŷₜ)²]
Penaliza erros grandes. Mais sensível a outliers. Mesma unidade da série.
MAPE
MAPE = (1/n)Σ|eₜ/yₜ|·100%
Erro percentual. Comparável entre séries. Problemático quando yₜ ≈ 0.

Backtesting — A Validação Real

Backtesting é o processo de testar uma estratégia ou modelo usando dados históricos passados. É a ponte entre teoria e prática em finanças quant.

1

Split Temporal (Walk-Forward)

Divide dados em treino (in-sample) e teste (out-of-sample). NUNCA use cross-validation aleatória em séries temporais — viola a ordem temporal.

2

Fitting e Previsão Rolling

Re-treina o modelo em janela deslizante, fazendo previsões 1-step-ahead. Simula o processo real de uso do modelo.

3

Avaliação Out-of-Sample

Calcula métricas de erro apenas no período de teste. Compara com benchmark (e.g., modelo naive, random walk).

4

Testes de Diagnóstico

Ljung-Box (autocorrelação dos resíduos), ARCH-LM (heterocedasticidade), normalidade dos resíduos.

⚠ Armadilhas do Backtesting

Look-ahead bias: usar informações do futuro durante o treino. Overfitting: modelo perfeito no treino, péssimo no teste. Data snooping: testar muitos modelos até achar um que funciona — o resultado é espúrio por sorte estatística.

WALK-FORWARD BACKTESTING · PREVISÃO VS REALIZADO
PYTHON
from sklearn.metrics import mean_absolute_error, mean_squared_error
import numpy as np

def walk_forward_validation(data, model_func, train_size, n_steps=1):
    predictions = []
    actuals = []
    
    for i in range(train_size, len(data) - n_steps + 1):
        train = data[:i]
        test  = data[i:i + n_steps]
        
        # Treina e prevê
        pred = model_func(train, steps=n_steps)
        predictions.extend(pred)
        actuals.extend(test)
    
    mae  = mean_absolute_error(actuals, predictions)
    rmse = np.sqrt(mean_squared_error(actuals, predictions))
    return {'MAE': mae, 'RMSE': rmse, 'preds': predictions}
Módulo 08 · Projeções & Risco

Value at Risk & CVaR

As principais métricas de risco de mercado usadas por bancos, fundos e reguladores do mundo todo.

VaR — Value at Risk

Perda Máxima Esperada

Com probabilidade (1-α), a perda não excederá VaR_α em um período t. Exemplo: VaR₉₅% = R$100k significa que há 5% de chance de perder mais de R$100k.

CVaR — Expected Shortfall

Perda Esperada na Cauda

Perda esperada dado que a perda excedeu o VaR. Mais conservador e coerente matematicamente. Exigido pelo Basileia III para risco de mercado.

CVaR / Expected Shortfall
CVaR_α = E[L | L > VaR_α] = (1/α)∫₀^α VaR_u du
Sempre CVaR ≥ VaR. CVaR é uma medida de risco coerente (satisfaz sub-aditividade). VaR não é coerente.

Métodos de Cálculo do VaR

MétodoAbordagemVantagensLimitações
ParamétricoAssume Normal: VaR = μ - z·σSimples, rápidoSubestima fat tails
HistóricoPercentil dos retornos históricosSem premissa distribucionalDepende do período histórico
Monte CarloSimula cenários futurosFlexível, capta não-linearidadesComputacionalmente intensivo
DISTRIBUIÇÃO DE PERDA/GANHO · VaR 95% E CVaR 95%
PYTHON
import numpy as np

def historical_var_cvar(returns, confidence=0.95, portfolio_value=1_000_000):
    alpha = 1 - confidence
    
    # VaR Histórico
    var_pct = np.percentile(returns, alpha * 100)
    var_brl = portfolio_value * abs(var_pct)
    
    # CVaR (Expected Shortfall)
    tail_returns = returns[returns <= var_pct]
    cvar_pct = tail_returns.mean()
    cvar_brl = portfolio_value * abs(cvar_pct)
    
    print(f'VaR {confidence*100:.0f}%:  {var_pct:.2%} | R$ {var_brl:,.0f}')
    print(f'CVaR {confidence*100:.0f}%: {cvar_pct:.2%} | R$ {cvar_brl:,.0f}')
    
    return var_pct, cvar_pct

Simulação de Monte Carlo

A ferramenta mais poderosa em finanças quantitativas: simular milhares de futuros possíveis.

Monte Carlo usa geração de números aleatórios para simular a distribuição de resultados possíveis. Em finanças, permite precificar derivativos complexos, calcular VaR, otimizar portfólios e projetar fluxos de caixa.

Aplicações em Finanças

Precificação

Opções Exóticas

Opções path-dependent (asiáticas, barreira, lookback) que não têm fórmula fechada são precificadas via Monte Carlo.

Risco

VaR & Stress Test

Simula distribuição de P&L do portfólio. Cenários extremos (tail risk) são capturados melhor do que com métodos paramétricos.

Planejamento

Projeção Financeira

Projeta receitas, custos e fluxos de caixa com incerteza. Calcula probabilidade de metas serem atingidas.

Precificação de Opção Call via Monte Carlo
C = e^(-rT) · E[max(S_T - K, 0)]
≈ e^(-rT) · (1/N) · Σᵢ max(Sᵢ_T - K, 0) para N caminhos simulados
PYTHON — PRECIFICAÇÃO DE CALL EUROPÉIA
import numpy as np

def mc_call_price(S0, K, r, sigma, T, N=100_000):
    # Gera preços terminais S_T
    Z = np.random.standard_normal(N)
    S_T = S0 * np.exp((r - 0.5*sigma**2)*T + sigma*np.sqrt(T)*Z)
    
    # Payoff da call
    payoffs = np.maximum(S_T - K, 0)
    
    # Desconta para valor presente
    price = np.exp(-r*T) * np.mean(payoffs)
    std_error = np.exp(-r*T) * np.std(payoffs) / np.sqrt(N)
    
    return price, std_error

# Exemplo: S=100, K=105, r=5%, σ=20%, T=1 ano
price, se = mc_call_price(100, 105, 0.05, 0.20, 1.0)
print(f'Call Price: R${price:.2f} ± {se:.4f}')

Derivativos & Precificação de Opções

Black-Scholes, Greeks, volatilidade implícita e estratégias com opções.

Black-Scholes — O Modelo Fundamental

Preço de Call Européia
C = S·N(d₁) - K·e^(-rT)·N(d₂)
d₁ = [ln(S/K) + (r + σ²/2)T] / (σ√T) | d₂ = d₁ - σ√T | N(·) = CDF da Normal padrão
Preço de Put Européia (Paridade Put-Call)
P = K·e^(-rT)·N(-d₂) - S·N(-d₁)
Put-Call Parity: C - P = S - K·e^(-rT). Garante ausência de arbitragem entre call e put.

PYTHON — BLACK-SCHOLES COMPLETO
from scipy.stats import norm
import numpy as np

def black_scholes(S, K, r, sigma, T, option='call'):
    d1 = (np.log(S/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    d2 = d1 - sigma*np.sqrt(T)
    
    if option == 'call':
        price = S*norm.cdf(d1) - K*np.exp(-r*T)*norm.cdf(d2)
        delta = norm.cdf(d1)
    else:
        price = K*np.exp(-r*T)*norm.cdf(-d2) - S*norm.cdf(-d1)
        delta = norm.cdf(d1) - 1
    
    gamma = norm.pdf(d1) / (S*sigma*np.sqrt(T))
    theta = (-(S*norm.pdf(d1)*sigma)/(2*np.sqrt(T)) - r*K*np.exp(-r*T)*norm.cdf(d2)) / 365
    vega  = S*norm.pdf(d1)*np.sqrt(T) / 100
    
    return {'price': price, 'delta': delta, 'gamma': gamma, 
            'theta': theta, 'vega': vega}

# Exemplo: call ATM, vol 20%, T=3m
result = black_scholes(S=100, K=100, r=0.05, sigma=0.20, T=0.25)
